ТЗ - это ты рассказываешь людям, какой фронт и бек  тебе надо сделать и как они должны работать
фронт - это формочки
бек - это вычисления и логика
например

окно создания нового персонажа
 - панель для выбора расы и пола
 — кликабельные иконки расы, описание
 - панель для выбора класса
 — кликабельные иконки класса, описание
 - панель для введения имени персонажа 

Вход по нажатию "Новая игра" в главном меню. 

Сперва выбор расы и пола одновременно. 
по кнопке "Далее" - переход на выбор класса
по кнопке Далее - переход на ввод имени, и кнопкой Готово - закончить. 
На каждой панели кнопка "Назад", на первой - "Назад в главное меню". 

Проверка имени - допустимы только русские буквы и цифры. 
Кнопки Далее неактивны, пока что-то не выбрано.

По результату заполнения формы - создавать запись в таблице Users, структура базы описана далее
формализовать свои юз-кейсы, чтоб они не допускали двусмысленности
чем круче они составлены, тем меньше будет WTF на этапе составления тз

Пример тз для окошка я тебе скинул. 
ТЗ для таблиц - список того, что хранится в таблице. 
ТЗ для хранимки - псевдокод, как она должна работать. 
ТЗ для класса - краткое описание, че он будет хранить и че он будет делать

ТЗ для таблиц

таблица Юзер
данные юзера из формы, автосоздающийся айди, дата создания юзера, статус (активен/забанен/удален)
ТЗ для хранимки

Заводим Юзера

проверяем, не занято ли имя
    если занято, выход
валидируем данные, если кривые, выход
записываем данные в табличку Юзерс
селектим этого юзера, получаем айди
возвращаем из хранимки айди новосозданного юзера

ТЗ для класса

ПанельВещейНаПолу

наследуется от JPanel, на нем будем рисовать вещь, что на полу. 

конструктор (место в ряду, иконка, кол-во)
    создает панель в контейнере в опред месте
    сует в приватное поле картинку, что будет отрисовываться
    сует в приватное поле кол-во предметов

переопределенный пейнт
    супер (г)
    рисует картинку из приватного поля
    сверху нее в нижнем углу рисует число предметов
